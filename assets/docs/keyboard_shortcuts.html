Within text areas and text fields

Aside from normal one-letter shortcuts for the actions, Helium also supports keyboard shortcuts inside a text area or a text field, which can be used while editing. The trick is to use the <Enter> button on your keyboard as a modifier key - similar to how you <Ctrl>+S to save a document on Windows (or <Cmd>+S on Mac), you press <Enter>+S to save a buffer inside Helium.

To view supported modified keys, you can press <Enter>+<Backspace> (use your thumb) inside any text field/area to bring up the complete list of keys recognized by that field/area. Most of the time it includes the same keys used for the actions, but it might also include special keys that can be useful e.g. <Enter>+D to delete the current line, <Enter>+V to paste.

Within each tab, while nothing is focused

Helium allows for keyboard navigation of the buffer being edited, when nothing is focused inside the tab i.e. no text field or area is focused.

  • <Space>: move the cursor down by a page
  • <Shift>+<Space>: move the cursor up by a page
  • U: move the cursor up by a page
  • J: move the cursor one line down
  • K: move the cursor one line up

Navigation in this way takes care of partial highlight (as compared to scrolling inside the text area, which requires you to tap at the desired location to place the cursor there and trigger the highlight).

Note: if the cursor is still within viewable area after any of these actions then there won't be any visible change - because a text area only updates its displayed area when the cursor is out of view. This is a limitation in Blackberry SDK.

Additionally, <Enter> key brings you to the most appropriate editable field inside the tab:

  • when the main text area is empty, this is the title field
  • when the main text area is not empty, this is the main text field
  • when Find & Replace is active, this is the find field
