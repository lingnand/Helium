Tabs and Buffers

At any time Helium manages a list of tabs which are views into buffers. A tab is a frontend that interacts with a buffer, whereas a buffer represents the actual data being edited i.e. what you see in the text area; and also data being saved to/loaded from files.

The distinction between tabs and buffers is necessary because Helium supports multiple tabs viewing the same buffer. This can be achieved in two ways:

  • use the Clone action inside any tab - this spawns a new tab that shares the same buffer with the previous tab
  • opening a file in a new tab which is already edited in another tab causes the new tab to share that existing buffer

Changes in a buffer are synced across all tabs linked to it. Additionally, settings accessed and changed via any tab's Properties action apply to the same buffer.

This means that you can for example:

  • edit a long document in one tab and refer back to its early parts in another tab
  • edit a script in one tab and run & view its output in another

Tips: a cloned tab shares the same working directory as the old one; so to open some other file in the same directory, you can clone the current tab and the Open action will navigate to that directory by default

Opening files

Helium supports opening multiple files at once. Simply use the Open action in any tab, choose the files you desire to edit and select Done. If there are more than one file chosen, the additional files will be put into new tabs just before the current one.

Each file is opened in a separate routine and progress is shown at the bottom of its respective tabs.

Saving files

Saving is done asynchronously i.e. every time you trigger the Save action, Helium copies the content in the current buffer, and starts a background routine that saves it to disk. Progress is displayed at the bottom of the tab in an non-interruptive way - this means that you can continue editing the document without having to wait for it to finish.

Highlight

Helium uses efficient algorithms to highlight documents in real-time. However, due to limitations on the amount of highlighted text that can be put into a text area, Helium uses partial highlight instead to keep highlight to the necessary minimum (the range of text around the cursor). If you still encounter performance problems, you can

  • turn off Enable Highlight for the given filetype, OR
  • decrease the range of the partial highlight

Filetype

Each filetype defines its own set of highlight rules, as well as having its own settings for highlight and run profiles. These settings can be accessed via Properties inside any tab (which shows settings of the filetype currently associated with the given buffer); they can also be accessed inside Settings-Filetypes.
